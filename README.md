# 📡 TG Mirror — Telegram Channel Mirror

Полное зеркалирование Telegram-каналов в ваш канал/группу с автоматическим
созданием тем (Topics/Forum).

> **Версия 3.2** — добавлен producer/consumer pipeline (параллельная закачка
> медиа вперёд), фильтр «обычных чатов», .env-поддержка, логирование в файл,
> dry-run режим, удаление истории из UI, прозрачный прогресс-бар, полная перерисовка
> экрана перед каждым меню. v3.1 исправила `ImportError UpdatePinnedMessageRequest`.

## ⚙️ Установка

### 1. Python 3.11+
```bash
python --version  # должно быть 3.11 или выше
```

### 2. Зависимости
```bash
pip install -r requirements.txt
```

### 3. (Опционально) .env
Рядом с `main.py` можно создать файл `.env`, чтобы API-ключи не хранились в `tg_mirror_config.json`:
```
TG_API_ID=12345
TG_API_HASH=abcdef...
```

### 4. API-ключи Telegram (бесплатно, 2 минуты)
1. Откройте https://my.telegram.org
2. Войдите своим номером телефона
3. Перейдите в «API development tools»
4. Заполните форму (название и платформа — любые)
5. Скопируйте **App api_id** и **App api_hash**

Программа спросит их при первом запуске и сохранит в `tg_mirror_config.json`.

## 🚀 Запуск

```bash
python main.py
```

При первом запуске:
1. Введите API ID и API Hash
2. Введите номер телефона (`+7XXXXXXXXXX`)
3. Введите код из Telegram
4. При необходимости — пароль 2FA

Сессия сохраняется в `tg_mirror_session.session`. Повторный вход не нужен.

## 🎯 Подготовка целевого канала

Для корректной работы с темами целевой канал должен быть **супергруппой с включёнными «Темами»**:

1. Создайте группу (или используйте существующую супергруппу)
2. Откройте «Управление группой» → «Темы» → Включите
3. Назначьте бот/аккаунт администратором с правами:
   - Отправка сообщений
   - Управление темами (Topics)
   - Закрепление сообщений (если нужно)

> Обычный канал тоже подойдёт — сообщения попадут в общий поток без тем.

## 📖 Главное меню

```
1 ▸  Запустить зеркалирование канала
2 ▸  Мои каналы и группы
3 ▸  Настройки
4 ▸  История синхронизаций
5 ▸  Сменить аккаунт
0 ▸  Выход
```

### Зеркалирование (пункт 1)

1. Выберите **источник** из списка (или введите @username / ID вручную)
2. Выберите **цель** — вашу группу/канал
3. Опционально укажите диапазон ID сообщений (или Enter = все)
4. Программа сама:
   - Читает все темы источника
   - Создаёт такие же темы в цели
   - Отправляет сообщения в правильные темы в хронологическом порядке
   - Сохраняет альбомы (группы фото) одной пачкой
   - Сохраняет цепочки ответов (reply chains), в том числе внутри тем
   - Закрепляет закреплённые сообщения

## 🔄 Возобновление

Если зеркалирование было прервано (Ctrl+C, ошибка сети) — просто запустите
снова с теми же каналами. Программа автоматически продолжит с места остановки.

Прогресс хранится в `sync_states/<src_id>_to_<dst_id>.json`.

- ✅ Цепочки ответов (reply threads), в том числе внутри тем

- ✅ Ссылки и inline-кнопки (текстовые ссылки)
- ⏭️ Стикеры / GIF — отправляются как файлы (ограничение API)
- ⏭️ Голосовые / видеосообщения — как документы

## 🕵️ Скрытность

Программа «не палит» ваш аккаунт перед источником:
- **Не накручивает просмотры** (нет вызовов `GetMessagesViewsRequest`)
- **Не отмечает сообщения прочитанными** (telethon `iter_messages` это не делает)
- **Не сигналит онлайн / typing** (нет `SetTyping`, нет `updateOnline`)
- **Скрытный режим** дополнительно отключает `catch_up` и `receive_updates` на клиенте

Для источника происходящее выглядит как просто «этот пользователь оффлайн».

## ⚡ Скорость

- **Pipeline producer/consumer**: пока отправляется сообщение N, в фоне скачиваются N+1, N+2 (буфер `prefetch_count`).
- **Параллельная закачка внутри альбомов** через `asyncio.gather` (ограничено `concurrent_downloads`).
- **Dry-run** перед реальным прогоном — увидеть, сколько будет отправлено и есть ли проблемы с доступом.

Обычные значения задержек (1.5–3 сек) можно уменьшить в Настройках, но будьте осторожны — слишком
агрессивные настройки вызовут FloodWait.

---

## ❗ Важно

- **Приватные каналы**: копирование работает если вы **являетесь участником**
- **Forward Protection**: защита от пересылки **не мешает** — данные читаются через MTProto API напрямую
- **FloodWait**: при ограничениях Telegram программа автоматически делает паузу и продолжает
- **Права в цели**: аккаунт должен быть администратором с правом управления темами
- **Файл сессии** (`*.session`) содержит авторизацию — храните его в безопасности

## 🐛 Частые проблемы

**`Could not find the input entity`** — канал не найден. Для приватных: убедитесь,
что вы являетесь участником.

**`ChatAdminRequiredError`** — нет прав в целевой группе. Назначьте аккаунт администратором
с правом управления темами.

**Темы не создаются** — убедитесь, что в целевой супергруппе включены «Темы» (Topics).

**FloodWaitError** — Telegram просит подождать. Программа делает паузу автоматически.
При частых ошибках увеличьте задержки в настройках.

**`ImportError: cannot import name 'UpdatePinnedMessageRequest'`** — была багом
ранних версий: запрос находится в `telethon.tl.functions.messages`, а не `channels`.
В версии 3.1 исправлено.
